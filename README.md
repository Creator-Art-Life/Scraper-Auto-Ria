# AutoRia Scraper (Ukrainska Versiya)

Цей проект є веб-скрейпером на Python, розробленим для збору даних про оголошення автомобілів з AutoRia (auto.ria.com), їх зберігання в базі даних PostgreSQL без дублікатів та щоденного створення дампів бази даних. Застосунок створений для високої ефективності, використовуючи асинхронні операції та кінцеву точку API для швидкого отримання даних, особливо для телефонних номерів. Він стійкий до змін інтерфейсу на веб-сайті AutoRia та містить зручні скрипти для запуску. Застосунок можна запускати як за допомогою Docker Compose (рекомендовано для продакшену), так і безпосередньо в середовищі Python (ідеально для розробки).

## Вирішена Бізнес-Задача

Цей проект повністю відповідає бізнес-задачі створення застосунку для періодичного скрапінгу платформи AutoRia (розділ вживаних автомобілів). Рішення перевершує початкові вимоги, реалізуючи розширені функції для швидкості, надійності та простоти використання.

**Ключові реалізовані вимоги:**

-   **Щоденний запланований скрапінг:** Застосунок запускається щодня в налаштований час, проходячи всі сторінки, починаючи зі стартової URL, для збору даних з кожного оголошення про автомобіль.
-   **Зберігання даних у базі даних PostgreSQL:** Усі зібрані дані постійно зберігаються в базі даних PostgreSQL.
-   **Відсутність дублікатів:** Система забезпечує цілісність даних, запобігаючи дублюванню записів, оновлюючи існуючі записи замість створення нових, якщо URL оголошення вже присутній.
-   **Щоденні дампи бази даних:** Щоденні дампи бази даних виконуються у визначений час, а файли дампів зберігаються в каталозі `dumps/` у кореневому каталозі проекту.
-   **Pip для управління пакетами:** `pip` використовується для управління всіма залежностями Python.
-   **`.env` для конфігурації:** Налаштування застосунку винесено та керуються через файл `.env`, що забезпечує гнучку та безпечну конфігурацію.
-   **Розгортання за допомогою Docker Compose:** Застосунок та база даних налаштовані для легкого розгортання за допомогою `docker-compose`, що спрощує налаштування виробничого середовища.
-   **Оптимізований збір даних:** Реалізовано з акцентом на швидкість та ефективність.
    -   **Асинхронні операції:** Використання асинхронних шаблонів для підвищення продуктивності під час збору даних.
    -   **Пряма кінцева точка API для телефонних номерів:** Замість повільніших, традиційних методів скрапінгу, застосунок використовує пряму кінцеву точку API для миттєвого отримання телефонних номерів, що значно прискорює процес збору даних.
-   **Повний збір полів:** Для кожного оголошення про автомобіль збираються всі вказані поля: `url`, `title`, `price_usd`, `odometer`, `username`, `phone_number`, `image_url`, `images_count`, `car_number`, `car_vin` та `datetime_found`.

## Структура Проекту

-   `docker-compose.yml`: Визначає сервіси Docker для бази даних PostgreSQL (`db`) та застосунку-скрейпера (`scraper`).
-   `.env`: Змінні середовища для підключення до бази даних, початкової URL AutoRia та часу планування. **(Вам потрібно створити цей файл вручну в кореневому каталозі проекту)**
-   `scraper/`:
    -   `Dockerfile`: Будує образ Docker для скрейпера, налаштовуючи середовище Python та встановлюючи залежності.
    -   `requirements.txt`: Перелік пакетів Python, необхідних для скрейпера.
    -   `main.py`: Основний скрипт Python, що містить логіку скрапінгу, взаємодію з базою даних та планування. Він організовує весь процес скрапінгу.
    -   `config.py`: Керує завантаженням змінних середовища та визначає загальні конфігурації.
    -   `core/`: Містить основні функції скрапінгу.
        -   `scraper_core.py`: Реалізує основну логіку скрапінгу, включаючи `collect_ad_urls_from_page` для вилучення посилань на оголошення (з адаптацією до різних структур інтерфейсу) та `parse_ad_page` для детального вилучення даних. Важливо, що він включає `get_phone_from_ria`, яка використовує прямий виклик API для отримання телефонних номерів, значно підвищуючи ефективність. Він розроблений з кількома резервними варіантами для пошуку елементів, що робить його стійким до незначних змін інтерфейсу на цільовому веб-сайті (наприклад, різні структури посилань, зміни в класах елементів).
    -   `database/`: Керує взаємодією з базою даних.
        -   `db_operations.py`: Обробляє підключення до бази даних PostgreSQL, збереження даних (з UPSERT для запобігання дублікатів) та отримання існуючих URL.
        -   `db_cleanup.py`: Надає функціональність для очищення всіх даних з бази даних.
    -   `file_operations/`: Містить утиліти для операцій з файловою системою.
        -   `file_writer.py`: (Якщо реалізовано для локального збереження/дампінгу файлів)
-   `dumps/`: (Створюється автоматично) Цей каталог буде зберігати щоденні файли дампів бази даних.
-   `clear_db.bat`: Зручний скрипт для очищення всіх даних з бази даних (для користувачів Windows).
-   `Makefile`: Надає набір команд для загальних завдань розробки та розгортання, спрощуючи операції.
-   `test_parser.py`: (Якщо присутній, містить тести для логіки парсингу)

## Поля Бази Даних

Таблиця `auto_ria_ads` в базі даних PostgreSQL буде зберігати такі поля:

-   `url` (TEXT): URL оголошення про автомобіль (унікальний ідентифікатор).
-   `title` (TEXT): Заголовок автомобіля.
-   `price_usd` (INTEGER): Ціна автомобіля в доларах США.
-   `odometer` (INTEGER): Пробіг автомобіля в кілометрах (наприклад, "95 тис. км" перетворено на 95000).
-   `username` (TEXT): Ім'я продавця.
-   `phone_number` (BIGINT): Контактний телефонний номер (наприклад, 38063...). Ефективно отримується за допомогою прямого виклику API.
-   `image_url` (TEXT): URL основного зображення автомобіля.
-   `images_count` (INTEGER): Загальна кількість зображень для автомобіля.
-   `car_number` (TEXT): Номерний знак автомобіля.
-   `car_vin` (TEXT): VIN автомобіля (ідентифікаційний номер транспортного засобу).
-   `datetime_found` (TIMESTAMP WITH TIME ZONE): Дата та час останнього збереження/оновлення даних автомобіля в базі даних.

Для зручних операцій Docker використовуйте наданий `Makefile`. Перейдіть до кореневого каталогу проекту у вашому терміналі та запустіть потрібну команду:

-   **`make start`**: Створює образи Docker (якщо не створені) та запускає сервіси бази даних та скрейпера у від'єднаному режимі (`-d`). Скрейпер буде безперервно скрапити AutoRia, і щоденні дампи бази даних будуть виконуватися в `DUMP_TIME`.
    ```bash

    ```
-   **`make logs`**: Відображає логи в реальному часі від сервісу `scraper`.
    ```bash
    make logs
    ```
-   **`make down`**: Зупиняє та видаляє всі сервіси Docker та мережі, визначені в `docker-compose.yml`.
    ```bash
    make down
    ```
-   **`make rebuild`**: Зупиняє, видаляє, а потім перестворює та перезапускає всі сервіси Docker. Корисно для застосування змін до Dockerfiles.
    ```bash
    make rebuild
    ```
-   **`make clean-db`**: Виконує скрипт `clear_db.bat` для очищення всіх даних з бази даних PostgreSQL.
    ```bash
    make clean-db
    ```

Застосунок автоматично створюватиме щоденні дампи бази даних PostgreSQL у час, вказаний в `DUMP_TIME` у файлі `.env`. Ці файли дампів будуть збережені в каталозі `dumps/` у корені проекту.

## Усунення Несправностей

-   **Змінні середовища не завантажено**: Переконайтеся, що ваш файл `.env` знаходиться в кореневому каталозі проекту та названий саме `.env`. Якщо запускаєте безпосередньо, перевірте, чи встановлені змінні середовища у вашій поточній сесії терміналу.
-   **Скрейпер не знаходить посилання / Проблеми з парсингом**: Логіка скрапінгу в `scraper/core/scraper_core.py` розроблена з кількома резервними варіантами та альтернативними селекторами для адаптації до незначних змін інтерфейсу AutoRia. Однак, значні структурні зміни на веб-сайті AutoRia можуть вимагати оновлення селекторів CSS або логіки парсингу в `scraper/core/scraper_core.py`. Використання прямих викликів API для телефонних номерів значно знижує ризик проблем з парсингом для цього конкретного типу даних.
-   **Проблеми з підключенням до бази даних**: Ще раз перевірте змінні середовища `POSTGRES_` у файлі `.env` та деталі підключення до PostgreSQL.

        -   `scraper_core.py`: Реализует основную логику скрапинга, включая `collect_ad_urls_from_page` для извлечения ссылок на объявления (с адаптацией к различным структурам интерфейса) и `parse_ad_page` для детального извлечения данных. Важно отметить, что он включает `get_phone_from_ria`, которая использует прямой вызов API для получения телефонных номеров, значительно повышая эффективность. Он разработан с несколькими резервными вариантами для поиска элементов, что делает его устойчивым к незначительным изменениям пользовательского интерфейса на целевом веб-сайте (например, различные структуры ссылок, изменения в классах элементов).
    -   `database/`: Управляет взаимодействием с базой данных.
        -   `db_operations.py`: Обрабатывает подключения к базе данных PostgreSQL, сохранение данных (с UPSERT для предотвращения дубликатов) и извлечение существующих URL.
        -   `db_cleanup.py`: Предоставляет функциональность для очистки всех данных из базы данных.
    -   `file_operations/`: Содержит утилиты для операций с файловой системой.
        -   `file_writer.py`: (Если реализовано для локального сохранения/дампирования файлов)
-   `dumps/`: (Создается автоматически) Этот каталог будет хранить ежедневные файлы дампов базы данных.
-   `clear_db.bat`: Удобный скрипт для очистки всех данных из базы данных (для пользователей Windows).
-   `Makefile`: Предоставляет набор команд для общих задач разработки и развертывания, упрощая операции.
-   `test_parser.py`: (Если присутствует, содержит тесты для логики парсинга)

## Поля Базы Данных

Таблица `auto_ria_ads` в базе данных PostgreSQL будет хранить следующие поля:

-   `url` (TEXT): URL объявления об автомобиле (уникальный идентификатор).
-   `title` (TEXT): Заголовок автомобиля.
-   `price_usd` (INTEGER): Цена автомобиля в долларах США.
-   `odometer` (INTEGER): Пробег автомобиля в километрах (например, "95 тыс. км" преобразовано в 95000).
-   `username` (TEXT): Имя продавца.
-   `phone_number` (BIGINT): Контактный телефонный номер (наприклад, 38063...). Эффективно получается с помощью прямого вызова API.
-   `image_url` (TEXT): URL основного изображения автомобиля.
-   `images_count` (INTEGER): Общее количество изображений для автомобиля.
-   `car_number` (TEXT): Номерной знак автомобиля.
-   `car_vin` (TEXT): VIN автомобиля (идентификационный номер транспортного средства).
-   `datetime_found` (TIMESTAMP WITH TIME ZONE): Дата и время последнего сохранения/обновления данных автомобиля в базе данных.

## Устранение Неполадок

-   **Переменные окружения не загружены**: Убедитесь, что ваш файл `.env` находится в корневом каталоге проекта и назван именно `.env`. Если запускаете напрямую, проверьте, установлены ли переменные окружения в вашей текущей сессии терминала.
-   **Скрепер не находит ссылки / Проблемы с парсингом**: Логика скрапинга в `scraper/core/scraper_core.py` разработана с несколькими резервными вариантами и альтернативными селекторами для адаптации к незначительным изменениям пользовательского интерфейса AutoRia. Однако, значительные структурные изменения на веб-сайте AutoRia могут потребовать обновления селекторов CSS или логики парсинга в `scraper/core/scraper_core.py`. Использование прямых вызовов API для телефонных номеров значительно снижает риск проблем с парсингом для этого конкретного типа данных.
-   **Проблемы с подключением к базе данных**: Еще раз проверьте переменные окружения `POSTGRES_` в файле `.env` и детали подключения к PostgreSQL.
